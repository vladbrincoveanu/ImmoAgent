One more question: Geocoding landmark hints

Listings with "near Kettenbrückengasse U-Bahn" — how to place the pin?

A

Pre-geocode at scrape time

When scraping, extract landmark hints and geocode them via Nominatim (OpenStreetMap). Store the resolved coordinates in MongoDB. Map just reads coordinates — instant, no runtime API calls.

B

Geocode at runtime (on demand)

Don't store coordinates — when the map page loads, look up each landmark hint via Nominatim in real-time. No schema changes, but adds latency and depends on external API availability.

C

Hybrid — pre-geocode common landmarks, runtime for rare ones

Build a local cache of known Vienna landmarks (U-Bahn stations, known streets, streetcar stops). Match hints against the cache first. Fall back to runtime Nominatim for anything not in the cache.

My recommendation: A — geocode once at scrape time. Nominatim is free and has no rate limits for light use. The geocoding already happens in your ViennaGeocoder. We'd just need to persist the resolved coordinates to MongoDB so they're available when the map loads.
